## What is Cloud?

### Traditional Approach (10-20 years ago)
- Companies bought physical servers from IBM, HP
- Created their own data centers
- **Data Center**: Place where servers are stored with proper:
  - Network connections
  - Temperature control
  - Power management
  - Security

### The Problem
- **Resource Waste**: Server with 100GB RAM + 100 CPU running 1 app
- Only used 1GB RAM + 1 CPU
- 99% resources wasted
- Very expensive approach

### Solution: Virtualization
- **Concept**: Create virtual servers on physical hardware
- **Benefit**: Deploy multiple applications on one physical server
- Instead of 15 servers → Use 1 server with virtual machines
- Much more cost-effective

### Cloud Definition
- Virtual machines accessible from anywhere in the world
- You don't know exact physical location
- Request resources and get them instantly
- Managed by system administrators

## Private Cloud vs Public Cloud

### Private Cloud
- **Who manages**: Your organization
- **Access**: Only your company employees
- **Control**: Complete control within organization
- **Tools**: OpenStack, VMware, Zen
- **Users**: Banks, sensitive data companies

### Public Cloud
- **Who manages**: Cloud providers (AWS, Azure, GCP)
- **Access**: Anyone with account
- **Control**: You control resources, they manage infrastructure
- **Examples**: Amazon Web Services, Microsoft Azure, Google Cloud

## Why Public Cloud is Popular

### 1. Maintenance Overhead Reduction (Main Reason)
- No need to manage data centers
- No dedicated IT team required
- 24/7 monitoring handled by provider
- No hardware maintenance
- No security patching worries

### 2. Cost Benefits
- Pay-as-you-go model
- No upfront hardware investment
- Scale up/down as needed

### 3. Easy Onboarding
- Create account and start using
- No complex setup required
- Perfect for startups and mid-scale companies

### AWS Service Growth
- Started with 20-30 services
- Now has 200+ services
- Continuously adding popular technologies
- Example: Kubernetes service (EKS)

## Why AWS is Popular

### 1. First Mover Advantage
- Pioneer in cloud computing
- Companies started cloud journey with AWS
- Once comfortable, they stay with AWS

### 2. Largest Market Share
- AWS → Azure → GCP (in order)
- More companies use AWS
- More job opportunities
- Better career prospects

### 3. Transferable Skills
- Cloud concepts are similar across platforms
- Learning one platform helps with others
- AWS knowledge opens doors to other clouds

## Cloud Repatriation (Moving Back to On-Premises)

### Reality Check
- Only 1-2% of companies move back
- Very small percentage
- Not a major trend

### Reasons for Moving Back
- Security concerns
- No real cost optimization achieved
- Didn't get expected benefits

### Why Most Stay in Cloud
- Startups can't afford data centers
- Maintenance complexity too high
- Cost of building infrastructure
- Need dedicated teams

**Conclusion**: Public cloud adoption continues to grow

## Creating AWS Account - Step by Step

### Prerequisites
- Valid email address
- Phone number
- Credit/Debit card (for verification)
- Address details

### Steps
1. **Go to AWS Sign-in Page**
   - Search "AWS sign in"
   - Click "Create new AWS account"

2. **Email Verification**
   - Enter email address
   - Choose "Root user"
   - Provide account name
   - Verify email with code

3. **Password Setup**
   - Create strong password
   - Follow AWS requirements

4. **Account Type**
   - Select "Personal" for learning
   - Provide full name
   - Enter phone number
   - Select country/region
   - Add address details

5. **Payment Information**
   - **Important**: Card needed for verification only
   - AWS won't auto-charge your card
   - Small verification amount (₹2 in India, $1 in US)
   - Money gets refunded
   - Avoid virtual cards - use real debit/credit card

6. **Additional Details**
   - PAN card info (if in India)
   - Tax information
   - Complete verification

### Important Notes
- **Free Tier**: Use free resources within limits
- **No Auto-Charging**: AWS won't deduct money automatically
- **Account Suspension**: Only if you don't pay actual charges
- **Security**: This account will be used for entire 30-day series
